Have you bought the 6th edition of "JavaScript: The Definitive Guide"? In this edition, David Flanagan devotes one chapter to jQuery.

Several years ago, I found jQuery. But it's little odd for me. I think .css is worse than .setStyle, jQuery object is too big, the first argument of each should not be index. And jQuery provides a few methods for non-DOM operation. I want to use more methods like Prototype's Enumerable.

But today, many programmers use jQuery. I think a famous interface is a good interface. And Underscore is a good complement to jQuery.

For me, Prototype is still not so bad. But jQuery + Underscore is also an acceptable choice.
